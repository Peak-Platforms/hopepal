👋 How to use this page

Select any week from the sidebar to edit it. Each lesson has five things that matter for the chatbot to work correctly — title, scripture, summary, keywords, and audio. Here is what each one does.

Title
The name of the teaching. Shown to the driver when a match is found. Replace the placeholder with your actual doc title.
Scripture
The primary verse for this week. Shown under the title in the lesson card. Format: Book Chapter:Verse — e.g. Jeremiah 29:11.
Summary
1–2 sentences describing what this teaching is about. Write it in plain language — this is what the chatbot searches when a driver types a query. The more specific and OTR-relevant, the better the matches.
Keywords
Specific words a driver might type or say — emotions, situations, OTR life. Type each one and press Enter. Examples: lonely, wife, 3am, argument, fired, kids, driveway. These are the primary search layer — add as many as fit.
Themes
Broader topic categories used as a fallback search. Keep these to 3–5 per lesson. Examples: identity, marriage, faith, purpose, forgiveness.
Audio URL
The direct track URL from AzuraCast after you upload the .m4a file. In AzuraCast go to Media → click the track → copy the stream/download URL. Paste it here and click Preview to confirm it plays before saving.
🔄 Weekly workflow
  1. Reformat your doc using the NotebookLM template
  2. Upload the doc to NotebookLM → generate Audio Overview → download .m4a
  3. Upload .m4a to AzuraCast Media library → copy the track URL
  4. Open this page → select the week → fill in title, scripture, summary, keywords
  5. Paste the AzuraCast URL → click Preview to confirm audio plays
  6. Toggle Publish to ON → click Save Changes
  7. Green dot appears in the sidebar — lesson is live in the chatbot

Lessons with Publish OFF are invisible to drivers until you turn them on. You can save a half-finished lesson and come back to it.
