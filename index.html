How It Works

Simple as a CB radio.
Secure as a vault.

Three steps. No accounts. No apps. Just a direct line between someone who needs hope and the person who can provide it.

01
📱
They open the chat
Anyone with the link can reach your ministry — from a rest stop, a hospital room, a recovery center, or a cab at 2am. No login, no signup.
02
🔐
Message encrypts instantly
Before it ever leaves their device, the message is AES-256 encrypted. The server never sees it in plain text. Ever.
03
🙏
You read it privately
Open your password-protected reader on any device. Messages decrypt locally — only you can read them. Respond on air, in person, or via text.

Privacy First

They'll send it because
they trust it.

For communities that don't trust systems — truckers, veterans, recovering addicts — visible privacy isn't a feature. It's the product.

🔐
AES-256-GCM End-to-End Encryption
Messages encrypt in the browser before leaving the device. The server stores only ciphertext.
🚫
Zero Plaintext Storage
No message is ever readable on the server or in transit. Only the reader with the key can decrypt.
👤
No Accounts Required
Senders need nothing — no login, no email, no profile. Just a message and a name if they want.
// Driver sends message
const plain = "Please pray for my family"
 
// Encrypted in browser
const cipher = AES256_GCM.encrypt(plain)
→ "a7Fd9xKp2mNq8rTv..."
 
// Server sees only ciphertext
POST /ministry-message
{ message: "a7Fd9xKp2mNq8rTv..." }
 
// Sidney's reader decrypts
✓ "Please pray for my family"
